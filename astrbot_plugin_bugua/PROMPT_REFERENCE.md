# 占卜 LLM Prompt 参考

## 完整 Prompt 示例

### 场景：用户问"期末考试运势"

#### System Prompt
```
你是古代易学大师，精通梅花易数与卦象解读。请根据卦象信息提供专业、深邃的占卜解读。
```

#### User Message
```
【用户问题】期末考试

【卦象信息】
本卦：豫卦
· 卦辞：利建侯行师。象曰：雷出地奋，豫。先王以作乐崇德，殷荐之上帝，以配祖。
· 关键词：喜悦、顺利、准备充分
· 运势：大吉

变卦（发展方向）：坤卦
· 关键词：包容、稳定、持续

【五行分析】
体卦五行属土
用卦五行属土
生克关系：比和（势均力衡，守成可利）

【要求】
1. 基于上述卦象与五行生克，分析用户问题的当前形势与未来发展
2. 阐述变卦所示的发展方向
3. 给出具体的建议或注意事项
4. 用词专业深刻，富有易学哲理
5. 限制在150字以内，直接给出解读内容（无需前缀说明）
```

#### 预期 LLM 回复
```
豫卦临门，象征时机已至。本卦雷动于地，预示学习动力充分，知识储备充足。土土相比，内外一心，学习状态稳定。变卦坤地厚德，暗示考试过程中需保持平稳心态，勿躁进。建议：夯实基础、相信积累，临场保持从容。考试当日早起调适心绪，以静制动。预期成果理想。
```

## Prompt 设计原理

### 1. System Role 设定

```
你是古代易学大师，精通梅花易数与卦象解读。
```

**为什么这样设计：**
- 场景化定位：让 LLM 进入"易学专家"的角色框架
- 避免 LLM 用现代心理学或科学主义解读（易学有其独特的逻辑体系）
- 引导输出风格向古代智慧倾斜，而非通俗鸡汤

### 2. 信息结构化

#### 【用户问题】
```
直接给出用户的核心疑问，帮助 LLM 聚焦于问题本身
示例：期末考试、感情、财运、工作等
```

#### 【卦象信息】
```
本卦（当前态势）：
  · 卦辞 - 古籍记载的含义（历史背景）
  · 关键词 - 现代关键语义（快速理解）
  · 运势 - 吉凶评级（一目了然）

变卦（发展趋势）：
  · 一般不给卦辞，只给关键词（简化信息）
  · 位置标记为"发展方向"（引导 LLM 理解时间维度）
```

**为什么分离本卦和变卦：**
- 本卦 = 现状（full info）
- 变卦 = 演变方向（关键词即可）
- 帮助 LLM 构建"因果链"逻辑

#### 【五行分析】
```
体卦/用卦属性 + 生克关系 + 结论

示例数据流：
  土 vs 土 → 比和 → "势均力衡，守成可利"
  火 克 木 → 体克用 → "主动有力，可制外境"
  水 生 木 → 用生体 → "用生体，外力扶助"
```

**为什么包含五行分析：**
- 五行是东方哲学的"密码"，支配生克制化
- 给 LLM 一个理性的分析维度，不只是象征理解
- 避免过于抽象或神秘的解读

#### 【要求】清单
```
1. 基于卦象与五行生克，分析形势与发展   ← 指导分析方向
2. 阐述变卦所示的发展方向              ← 强调时间维度
3. 给出具体建议或注意事项              ← 实用性要求
4. 用词专业深刻，富有易学哲理          ← 风格约束
5. 限制150字以内，直接给出内容         ← 格式和长度

→ 这些要求直接对应提示工程的常见模式
```

## 核心词汇说明

### 易学术语维度

| 术语 | 含义 | Prompt 中的应用 |
|-----|------|----------------|
| 本卦 | 当前态势 | 现状分析的基准点 |
| 变卦 | 演变结果 | 未来趋势指示 |
| 体卦 | 主动方/我方 | 主体利益所在 |
| 用卦 | 被动方/外力 | 环境因素所在 |
| 五行 | 生克制化的规律 | 理性分析维度 |

### 五行生克关系

```
生（顺畅）：金→水→木→火→土→金（循环）
克（制约）：金克木、木克土、土克水、水克火、火克金（循环）

在 Prompt 中的应用：
- 用生体：外界支援，吉利
- 体生用：自我消耗，不利
- 比和：平衡，中平
- 体克用：主动优势，吉利
- 用克体：被动劣势，不利
```

## 调优指南

### 如果 LLM 回复过于通俗

**调整方式：**
```diff
- 你是古代易学大师，精通梅花易数与卦象解读。
+ 你是精通《易经》与《梅花易数》的古代哲学家。
+   请从宇宙哲学、阴阳变化、天人合一的角度进行深度解读。
+   避免现代心理学或成功学的表述方式。
```

### 如果 LLM 回复过于冗长

**调整方式：**
```diff
- 限制在150字以内，直接给出解读内容（无需前缀说明）
+ 用不超过100字的精炼语言，直接给出解读（无任何前缀）
```

### 如果 LLM 忽视五行分析

**调整方式：**
```diff
- 【五行分析】
+ 【五行深度分析】
  体卦五行属{ti_elem}，【关键解释："土"代表稳定与包容】
  用卦五行属{yong_elem}，【关键解释："火"代表热情与推进】
  生克关系：{relation}，具体含意是{desc}
```

### 如果 LLM 输出缺乏针对性

**调整方式：**
```diff
+ 用户具体问题是"{topic}"，所有解读必须直指此问题。
+ 解读中至少包含3条针对"{topic}"的具体建议。
```

## 使用建议

1. **保持结构一致性**
   - 不要随意改变 【】 的顺序
   - 确保 user message 中总是包含"用户问题"作为首要上下文

2. **定期测试**
   - 用不同的卦象组合测试 LLM 输出
   - 检查是否有"幻觉"（编造不存在的易学概念）

3. **监控生克关系**
   - 五行生克是逻辑基础，LLM 必须正确理解
   - 错误样例：说"土克水"（其实是水克火）

4. **保留降级方案**
   - 本地模板 (_ADVICE_TABLE) 始终作为备选
   - 当 LLM 返回异常时自动转换

## 技术细节

### 如何调用这个 Prompt

```python
system_prompt, user_message = DivinationService._generate_llm_prompt(
    topic="期末考试",
    ben_data={"name": "豫卦", "description": "...", ...},
    tiyong={"ti_elem": "土", "yong_elem": "土", "relation": "比和", ...},
    bian_data={"name": "坤卦", "keyword": "包容"},
    time_info={...}  # 农历时间等信息
)

# 返回结果是元组
# system_prompt = "你是古代易学大师..."
# user_message = "【用户问题】期末考试\n【卦象信息】..."
```

### 在 LLM 调用中使用

```python
llm_resp = await context.llm_generate(
    messages=[{"role": "user", "content": user_message}],
    system_prompt=system_prompt,
    temperature=0.7,          # 控制创意程度（0=确定，1=随机）
    max_tokens=200,           # 最多生成200个token
    tools=None                # 无需函数调用
)
```

## Q&A

**Q: 为什么不用中文 prompt？**
A: 本身就是中文 prompt。框架支持多语言，但易学概念用中文表达最准确。

**Q: 为什么限制在150字？**
A: 占卜回复需要简洁有力，150字足以说透关键要点，过长反而削弱易学的"一言以蔽之"的精妙。

**Q: 能否让 LLM 自动生成卦辞？**
A: 不建议。原卦辞来自《周易》经文，具有权威性。LLM 生成的卦辞可能"幻觉"。

**Q: 如何处理 LLM 诱导用户迷信？**
A: 在 prompt 中加约束：
```
请在解读中包含"这是易学参考，最终决策权在用户"的风险提示。
```
